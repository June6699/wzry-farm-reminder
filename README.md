# 王者农场自动提醒

一个面向《王者荣耀》农场玩法的本地辅助工具，重点解决这些问题：

- 什么时候播种、浇水、收获
- 周末双倍怎么卡时间
- 防偷窗口怎么避开风险
- 钱包余额和流水怎么自动记
- 偷别人菜、被别人偷菜怎么进账
- 多格子规划后，怎么继续手动改单格

当前版本已支持 `Windows` 本地运行，技术形态是：

- `Python 3.11`
- `SQLite`
- `JSON 规则与配置`
- `本地 Web UI`

## 部署流程

### 1. 推荐环境

推荐和当前开发环境尽量保持一致：

- `Python 3.11.0`
- `pip 25.0.1`
- `setuptools 75.8.2`
- `wheel 0.45.1`

说明：

- 当前项目运行时只依赖 Python 标准库。
- 也就是说，项目本身**没有额外第三方运行依赖**。
- 这里仍然建议把 `pip / setuptools / wheel` 统一好，方便后续打包和分发。

### 2. 安装 Python

推荐安装：

- `Python 3.11.0 x64`

安装后在终端验证：

```powershell
py -3.11 --version
```

或：

```powershell
python --version
```

### 3. 安装/统一包工具版本

在项目目录中执行：

```powershell
py -3.11 -m pip install --upgrade pip==25.0.1 setuptools==75.8.2 wheel==0.45.1
```

可选验证：

```powershell
py -3.11 -m pip --version
py -3.11 -m pip show setuptools
py -3.11 -m pip show wheel
```

### 4. 当前版本无需额外第三方运行包

也就是说，以下命令当前**不需要再额外执行任何业务依赖安装**：

```powershell
py -3.11 -m pip list
```

如果后续项目增加打包、邮件提醒、站点部署等能力，再补新的依赖列表。

### 5. 启动项目

方式 1：

```powershell
py -3.11 main.py
```

方式 2：

```powershell
run_app.bat
```

启动后会打开本地页面：

```text
http://127.0.0.1:8765/
```

### 6. 恢复默认数据

把所有配置、日志、钱包流水、规划状态恢复为默认空白：

```powershell
py -3.11 reset_to_defaults.py
```

或：

```powershell
reset_to_defaults.bat
```

这个脚本会做几件事：

- 尝试关闭本地服务
- 清空业务数据表
- 重建默认配置
- 保留规则文件

## 当前功能

### 1. 钱包与流水

已支持：

- 当前余额
- 今日收入
- 今日支出
- 今日净收益
- 最近流水列表

流水会记录：

- 买种子支出
- 收获收入
- 偷别人菜收入
- 被偷菜损失

### 2. 规划系统

已支持：

- 选择作物并生成规划
- 现在立刻种的收益/风险判断
- 最早防偷方案
- 最优周末双倍方案
- 周末双倍 + 防偷叠加方案
- 一键把规划应用到已开格子

### 3. 农场编辑

已支持：

- 按格子解锁
- 2 x 2 地块布局
- 每块地 4 x 3 格
- 单格显示作物与剩余时间
- 编辑范围：
  - 全部已开格子
  - 某一整块地
  - 某一个格子
- 手动改单格/整块地/全部范围的：
  - 作物
  - 状态
  - 开始时间
  - 收获时间
  - 备注
- 手动开始种植
- 登记浇水
- 登记收获
- 清空规划

### 4. 浇水机制

当前版本已支持：

- 手动输入本次浇水减少的分钟数
- 更新收获时间
- 更新剩余时间
- 写入活动记录

当前设计思路：

- 允许用户“提前浇水”或“不按推荐时机浇水”
- 由用户明确输入本次扣掉的冷却时间
- 系统据此重算剩余时间和计划

### 5. 偷菜进出账

已支持两种方向：

- 别人偷了我的菜
- 我去偷了别人的菜

规则：

- 被偷菜时，`偷菜人` 必填
- 偷别人时，`受害者` 可空
- 两边都支持作物搜索

### 6. 统计页

已支持：

- 时间分布散点图
- 月份筛选
- 事件类型筛选
- 种植、浇水、收获、被偷菜、偷别人菜用不同颜色区分
- 右侧统计摘要

图表规则：

- 横轴：日期
- 纵轴：一天中的时间
- 悬浮：显示事件详情

### 7. 设置页

已支持可视化配置：

- 昵称
- 游戏 ID
- 等级
- 小摊加成
- 已开格子数
- 各类提前提醒时间
- 周末目标时间
- 防偷开关
- 防偷时间窗口

## 关键逻辑

### 1. 为什么是“已开格子数”

当前项目已经按你的要求改成：

- 不再用“已开几块地”作为真实状态
- 改成“已开格子数”
- 地块只是布局容器
- 真正参与规划和编辑的是格子

### 2. 为什么规划和农场编辑要分开

原因是两者职责不同：

- `规划页` 负责算“推荐方案”
- `农场页` 负责在方案基础上继续手动微调

所以现在流程是：

1. 在规划页选作物、算方案
2. 一键应用到已开格子
3. 去农场页改单格、改单块、改时间
4. 手动确认开始种植
5. 后续登记浇水和收获

### 3. 规划状态和已种植状态的区别

- `已规划`：只是把方案布置好了，还没有确认已经在游戏里种下
- `已种植`：用户已经手动确认开始种植，才开始真正按剩余时间倒计时

这也是为什么：

- 规划时间过了，不会直接当成“已成熟”
- 会显示“计划时间已过，待手动开始”

### 4. 为什么当前版本不用额外 pip 包

因为当前版本优先追求：

- 轻量
- 便携
- 方便分享
- 少环境问题

所以目前核心依赖都用标准库完成：

- `http.server`
- `sqlite3`
- `json`
- `pathlib`

## 用法建议

推荐日常使用顺序：

1. 先在设置页填好昵称、等级、小摊加成、已开格子数
2. 去规划页算一份方案
3. 应用到已开格子
4. 去农场页按需要改单格作物或时间
5. 真正在游戏里种下后，点“手动开始种植”
6. 中途浇水就点“登记浇水”
7. 收获时点“登记收获”
8. 如果有偷菜行为，就去偷菜登记页补账
